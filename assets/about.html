Project Instructions:

  • Frontend -
    • A simple form for entering scrape requests which will will inclunde basic params: URL, max depth, max pages.
    • A display of the actual jobs with as much relevant information.
  • Backend -
    • An API service for the frontend to communicate with. It should implement eirther REST API or a well structured API over Websockets.
    • A crawler service that handles the actula crawling with the following specs:
      • Use the BFS algorithm. BFS here means that all items of depth N. Finish processing before any items from depth N+1 start.
      • Stop crawling a job when it reaches maxDepth or maxPages whichever comes first.
      • For each page, save at least the title, depth, URL, links.
  • Build for horizontal scale. Pure in-memory solutions will not be accepted.
  • Make the UI update in real time.
